🚀 1. The Quick Start (All-in-One Local Launch)
If you want to spin up both the FastAPI Backend Server and the Vite Web Dashboard instantly, you can use your pre-configured batch script from the root c:\ciro directory:

powershell
# Double-click or run from PowerShell root:
.\start_demo.bat
This starts the Python agentic engine on http://localhost:8000 and the React web dashboard on http://localhost:5173 automatically.

🖥️ 2. Web Dashboard (Vite + React)
The Web Dashboard resides in c:\ciro\web-dashboard. Open a terminal and run:

powershell
# Navigate to the dashboard directory
cd c:\ciro\web-dashboard
# Install dependencies (only required once)
npm install
# Start the local development server (Vite)
npm run dev
# Build the production bundle (to test compilation and types)
npm run build
# Run a local preview of the production build
npm run preview
📱 3. Mobile App (Expo React Native)
The Mobile App resides in c:\ciro\mobile-app and uses Expo. Open a new terminal and run:

powershell
# Navigate to the mobile app directory
cd c:\ciro\mobile-app
# Install dependencies (only required once)
npm install
# Start the Expo Metro Bundler
npm start
📲 Testing the Mobile App:
Once you run npm start, Expo will open an interactive menu in your terminal. You can run it on your desired platform:

Test on Real Device (Best for Demo): Download the Expo Go app on your iOS App Store or Android Play Store. Scan the QR Code that prints in your terminal. The app will sync instantly over local Wi-Fi!
Test on iOS Simulator (requires macOS / Xcode): Press i in the terminal or run:
powershell
npm run ios
Test on Android Emulator (requires Android Studio / AVD): Press a in the terminal or run:
powershell
npm run android
Test via Web Browser Preview (Vite-style React Native Web preview): Press w inside the Expo start session.
Run Unit Tests (Jest):
powershell
npm test
🚒 4. Re-running the Emergency Contacts Routing Test
To demonstrate the newly integrated city-specific emergency routing to your teammate or judges, run our test script from the backend folder:

cd c:\ciro\backend
